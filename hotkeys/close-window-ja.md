# prefix+x — 選択中リージョンの現在のウィンドウを閉じる

> **言語** · [中文](close-window-zh.md) · [English](close-window-en.md) · **日本語**

[← 設定ヘルプに戻る](../settings-help-ja.md)

`prefix+x` は選択中の region の現在の 1 ウィンドウを閉じます。**アプリは再起動せず**、そのウィンドウだけを閉じます。

> デフォルト chord: `x` · 設定項目: `close_window` · 「設定 → ホットキー → リージョン操作」で変更

---

## 発動の流れ

```
1. （任意）prefix+q N    で region を選択。選ばない場合はデフォルトで region 0
2.        prefix+x       で当該 region の現在のウィンドウが WM_CLOSE を受け取る
```

---

## クローズのセマンティクス

標準の Win32 `WM_CLOSE` メッセージを通します:

- **非同期**: 送信後すぐに戻り、アプリ自身がいつ閉じるかを決定する。
- **アプリが拦截可能**: 未保存のドキュメントアプリ（Office / VSCode / メモ帳）は「保存しますか？」ダイアログを表示する。ユーザーがキャンセルを選ぶとウィンドウは閉じない。
- **アプリが無視可能**: ごく少数のアプリ（一部のゲーム / デーモンプロセス）は `WM_CLOSE` に応答せず、`prefix+x` を押しても「反応なし」のように見える。

> アプリプロセスを強制終了したい場合（`WM_CLOSE` を経由しない）、タスクマネージャー / `Stop-Process` を使ってください。gmux はそのような破壊的能力を提供しません。

---

## クローズ後に何が起きるか

- 当該 region の一時フルスクリーンマークがクリアされる。
- エンジンは自動的に region に次のウィンドウを配置しない。
- 「次の Chrome ウィンドウを自動的に上に出したい」: `prefix+f` を使用。
- 「レイアウト全体をもう一度配置したい」: `prefix+r` を使用。
- region 自体は消えない（split で生成された region が一部のクリーンアップパスで戻る場合を除く）。

---

## 拒否条件

| 状況 | 動作 |
|---|---|
| 現在使用可能なレイアウトがない | サイレント |
| selected_region に対応する位置にウィンドウがない | サイレント; 同時に当該 region の stale なフルスクリーンマークをクリア |
| ウィンドウが既に死んでいる | サイレント |

---

## 誤って閉じてしまったら

`prefix+x` はアプリ自身のクローズフローを通すため、**アンドゥはありません**。

- アプリが「最近閉じたものを開く」をサポートしていれば（ブラウザの Ctrl+Shift+T、IDE の reopen closed）、アプリ標準機能で復旧してください。
- ドキュメントアプリの場合、「保存しますか？」ダイアログでキャンセルを押せば誤クローズを回避できます。
