# prefix+? — 打开帮助 / 速查表窗口

> **语言** · **中文** · [English](open-help-en.md) · [日本語](open-help-ja.md)

[← 返回设置页帮助](../settings-help-zh.md)

`prefix+?` 打开设置窗口并跳到「帮助」页（速查表）。

> 默认 chord：`?` · 配置项：`open_help` · 在「设置 → 快捷键 → 布局导航」改

---

## 默认键的实现细节

显示给用户的是 `?`，但引擎在内部把它**等价于 `/` 键**：

- 不需要按 Shift。
- 在中文键盘 / 英文键盘 / 日文键盘上都用 `/` 键位（一般在 `.` 旁边）。
- 设置「快捷键」页改键时把 `?` 选给其它动作不会冲突，因为引擎会做 shift-OEM 的别名归一化。

---

## 行为

按 `prefix+?` 后：

- 如果设置窗口已经打开 → 跳到「帮助」标签页，把窗口提到前台。
- 如果设置窗口没开 → 打开它并停在「帮助」页。
- 没有任何视觉副作用 / 不动当前布局 / 不改 selected_region。

---

## 帮助页显示什么

「帮助」页是动态生成的速查表：

- **当前 prefix**（修改 prefix 后立即反映）。
- **所有绑定**：每条带 `prefix → <key>` 和绑定描述。
- **所有内置动作**：每条带 `prefix → <chord>` 和动作中文说明。
- 文档与支持链接：在线文档、报告问题、关于 gmux。

绑定 / chord 的修改会通过 `config-reloaded` 事件自动同步刷新这一页。

---

## 与 F1 的关系

设置窗口聚焦时按 `F1` 等价于 `prefix+?`。

例外：在 **「关于」** 和 **「开源软件许可证」** 页 F1 不响应（避免阅读时被打断）。这两页要看帮助请点侧栏。

---

## 拒绝条件

| 情况 | 行为 |
|---|---|
| 启动时 callback 未安装（仅独立 CLI / 测试构建） | 静默无操作 |

正常 Tauri shell 构建下永远会成功打开。
