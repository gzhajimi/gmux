# prefix+n — 切到下一个布局

> **语言** · **中文** · [English](next-layout-en.md) · [日本語](next-layout-ja.md)

[← 返回设置页帮助](../settings-help-zh.md)

`prefix+n` 在当前匹配的布局列表里向后走一格。

> 默认 chord：`n` · 配置项：`next_layout` · 在「设置 → 快捷键 → 布局导航」改

---

## 它在哪些布局之间循环

- 范围 = 当前接入屏匹配的所有绑定（也就是 `prefix+w` OSD 列出的那一批）。
- 顺序 = 「布局」页左栏的拖拽顺序。
- 末尾回到开头（环游）。

> 当前布局未在该列表里（比如刚经历显示器热插拔）时，`prefix+n` 会从列表的第一个开始。

---

## 不在范围内的布局

下列绑定**不会**被 `prefix+n` 经过：

- 没有任何预案与当前接入物理屏匹配。
- 免费版下被锁定的绑定（超过 9 个，或多屏预案在免费版下）。

---

## 触发流程

直接按 `prefix+n`，无需第二键。

可以快速连按：`prefix+n n n n …` 在所有匹配的布局间快速走。每次按下都会走一格并立即应用对应的布局。

---

## 与 last_layout (`prefix+l`) 的区别

| 动作 | 语义 |
|---|---|
| `prefix+n` | 单调环游：A → B → C → A → B → …，永远向后 |
| `prefix+p` | 单调环游：A → C → B → A → C → …，永远向前 |
| `prefix+l` | toggle：跳到「上一次执行过」的布局，A → B → A → B → …（按访问历史） |

两套不冲突：`n/p` 适合「想看看每个布局长什么样」式的浏览，`l` 适合「在两个高频布局间来回切」。

---

## 拒绝条件

| 情况 | 行为（开启静默提示时弹文案） |
|---|---|
| 没有任何匹配当前屏的布局 | 静默 / 弹 `step_no_layout` |
